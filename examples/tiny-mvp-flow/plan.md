# Product Plan

## Product

Pocket Tasks, a local-first browser task list for a solo builder.

## Target Users

Solo builders who need a tiny task board while shaping an MVP.

## MVP

Users can add a task, mark it complete, delete it, and keep tasks after a browser reload.

## Current Stage

Tiny MVP

## Build Style

Web app

## Non-goals

No auth, backend, sync, collaboration, analytics, or paid services.

## Constraints

Use local storage only. Keep the UI simple enough to validate in one browser smoke test.

## Success Criteria

One local task flow works: create, complete, delete, reload, and see persisted tasks. Build or smoke validation passes.

## Current Focus

Build the smallest local task list MVP.
